<h1>
 Бекенд мессенджеру
</h1>
<h4> Автори: Подлипський Гліб ІМ-43</h4>
<h4>
  Технологія для API : Nest js + Typescript
  БД: Postgres + Typeorm
  Фреймворк тестування: Jest
</h4>
<p>
  В рамках виконання курсової роботи я вирішив розробити бекенд месенджеру.
  У нас є такі сутності як: користувач, чат, учасник чату, повідомлення, реакція, та тип реакції.
</p>

<p>
 Будь які операції з нашим бекендом може проводити лише авторизований юзер.
 Для того щоб авторизуватися юзер має пройти логін, або зареєструватися.
 Якщо юзер ввів валідні дані під час авторизації то бекенд поверне йому токен jwt. 
  Цей токен буде в подальшому використовуватися для
 підтвердження юзера під час виконання операцій.
</p>
<p> 
  Після авторизації юзер може створити чат, ввівши для цього тег, назву та описа(опціонально).
  В разі успішного створення чату юзер автоматично отримає в ньому роль супер адміна, що дасть дозвіл майже на будь-які дії в цьому чаті.
  До існуючого чату юзер може додати інших користувачів.
  Також якщо його роль в чаті дозволяє він може банити інших юзерів, видаляти та редагувати чужі повідомлення. 
</p>
<p>
  Будучи учасником чату юзер може писати в ньому повідомлення.
  Для цього він має тільки вказати текст повідомлення.
  Також юзер зможе реагувати на вже створенні повідомлення.
  Для цього він має обрати повідомлення та вказати тип реакції. 
</p>
